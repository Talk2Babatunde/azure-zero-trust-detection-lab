# 08 - Incident Investigation: The SOC Analyst Workflow

## 🕵️ Investigation Overview
Once a detection rule triggers, the transition from **Alert** to **Incident** begins. This project demonstrates the full lifecycle of a security incident within Microsoft Sentinel, focusing on how an analyst correlates disparate logs to reconstruct an attacker’s timeline.

---

## 📈 The Investigation Lifecycle

### 1. Triage & Scoping
When the "Brute Force to Key Vault Access" incident triggered, my first step was to scope the impact:
- **Entities Involved:** User account `Admin-Babatunde`, Source IP `203.0.113.5`, and Resource `kv-prod-secrets`.
- **Severity Assessment:** Classified as **High** due to the successful access of a production Key Vault following an authentication anomaly.

### 2. Deep Dive: Timeline Reconstruction
Using the **Sentinel Investigation Graph**, I reconstructed the adversary's steps:
1.  **09:00 AM:** 150 failed RDP attempts detected from IP `203.0.113.5`.
2.  **09:12 AM:** A single successful login (Event 4624) recorded from the same IP.
3.  **09:15 AM:** The user account performed a `List Keys` operation on the Key Vault.
4.  **09:17 AM:** A `SecretGet` operation was recorded, indicating potential data exfiltration.

### 3. Root Cause Analysis (RCA)
The investigation revealed that the `Admin-Babatunde` account did not have **Multi-Factor Authentication (MFA)** enforced, and the **Network Security Group (NSG)** allowed RDP traffic from any source IP (0.0.0.0/0). This combination allowed the brute-force attack to succeed.

---

## 🛠️ Tools Used During Investigation
- **Sentinel Investigation Graph:** For visual relationship mapping between the IP and the VM.
- **Log Analytics (KQL):** Used to "hunt" for lateral movement by checking for other logins from the same malicious IP across the Hub-and-Spoke network.
- **Azure Activity Logs:** To verify if any unauthorized RBAC changes were made during the session.

---

## 📉 Final Disposition: True Positive
This incident was confirmed as a **True Positive (TP)**. 

### Recommended Remediation (The "Next Steps"):
1.  **Immediate:** Revoke all active sessions for `Admin-Babatunde` and reset the password.
2.  **Short-term:** Update the NSG to restrict RDP access to the Company VPN IP range only.
3.  **Long-term:** Enforce **Conditional Access Policies** requiring MFA for all administrative actions in the Azure tenant.
